CONTEXT
배경 · 환경

왜 지금, 왜 사내 도구인가?

외부 솔루션 없이도, 지금 가진 MS 도구와 AI만으로 운영 품질을 끌어올릴 수 있는지를 검증한 파일럿입니다.

기본 방향

  • 보안·권한 검토 없이 바로 쓸 수 있는 방식
  • 추가 라이선스/구매 없이 MS365 중심 구현
  • AI는 “관점/초안 제공자”로만 활용
“새 시스템 도입”이 아니라, 현재 도구를 조합·정리하는 데 초점을 맞췄습니다.

Why now?

  • 교육 안내·보고 메일을 연간 다수 수기 작성
  • 담당자·과정마다 서식·문체가 달라 검수 공수 증가
  • 해킹대회·세미나 확대 → 오류·누락 리스크 커짐

사내 도구 중심 운영

  • MS365 / Outlook / Teams / Forms / Power Automate 기반
  • 외부 AI·툴 없이도 운영 가능한 구조
  • GitHub Copilot 등 사내 허용 AI로 보조 활용

PROBLEM · GOAL
현황 · 운영원칙

현황 · 문제 인식 · 개선 목표

“많이 바쁘지만, 늘 비슷한 일을 반복한다”는 현장의 피로를 줄이는 것이 출발점이었습니다.

추진 배경

  • 교육 안내·보고 문서 다수, 담당자별 형식 제각각
  • 형식 검수·수정에 건당 20~30분 추가
  • 대규모 행사 시 오류·재작성 공수 발생

현장의 문제 인식

  • 과정명·일시·장소 매번 재입력 → 복붙 오타
  • 폰트·여백·문체 불일치로 가독성 저하
  • 출석·결과·보고 흐름이 끊겨 재작성 반복

개선 목표 (운영 원칙)

  • 단순 Simple – 한 번 입력해 여러 단계 활용
  • 일정 Steady – 서식·문체 기준 통일
  • 연결 Connected – 공지→출석→보고 흐름 유지
이번 파일럿은 이 세 가지 원칙을 실제 업무에 적용해 본 첫 시도입니다.

OVERVIEW
전체 구조

이번에 직접 개선한 3가지

오늘 데모는 “메일 템플릿 · 안내 사이트 · PIN 출석” 세 가지 개선 흐름으로 보시면 됩니다.

한 줄로 요약하면

  • 공지 메일: HTML 템플릿으로 통일
  • 개강 안내: 웹·메일 한 번에 관리
  • 세미나 출석: PIN + 자동 기록 구조
세 가지 모두 한 번 입력한 정보가 여러 단계에 재사용되도록 설계했습니다.

① HTML 템플릿 (메일·인쇄)

  • 공지 메일·인쇄물을 단일 HTML 템플릿으로 생성
  • Outlook·EP 메일 호환, 레이아웃 깨짐 최소화
  • 운영 원칙: 단순 · 일정

② 개강 안내 사이트

  • 기존 안내문을 웹 페이지로 전환
  • 외부 강사·내부 구성원이 같은 정보 기준 사용
  • 운영 원칙: 일정 · 연결

③ PIN 기반 세미나 출석

  • Class별 PIN 코드로 공용 PC에서 출석 입력
  • MS 연동으로 출석 시간·명단 자동 기록
  • 운영 원칙: 단순 · 연결

DEMO 1 · HTML TEMPLATE
메일/인쇄 안내

개선 1 – HTML 템플릿 (메일·인쇄)

“메일·인쇄물은 템플릿 1종으로 통일” → 작성 시간은 줄이고, 품질은 일정하게.

Before · 수기 메일/이미지

  • 과정마다 Word/PowerPoint 양식을 새로 작성
  • EP·Outlook에서 이미지·줄바꿈 깨짐
  • 작성+검수 포함 1건당 약 35분 소요
담당자·과정이 바뀔 때마다 처음부터 다시 만드는 느낌이 반복되었습니다.

After · HTML 템플릿 1종 통일

  • 과정명·일시만 입력하면 안내문이 자동 생성
  • 버튼·색상·폰트 통일 → 운영 품질 일정화
  • 제작 시간 약 7분, 약 80% 단축

※ 이미지는 images/mail_template_sample.png를 실제 캡처로 교체해 사용합니다.

DEMO 3 · ATTENDANCE
출석 흐름

개선 3 – 세미나 출석 흐름

종이 서명 대신 “PIN + 자동 기록” → 출석 누락과 수기 정리를 한 번에 줄였습니다.

운영 관점의 변화

  • “출석표 회수 → 엑셀 입력” 루틴 제거
  • 출석 시간/인원/세션별 참여율이 자동 남음
  • 재작성/재확인에 쓰던 시간을 피드백 정리로 전환
출석만 자동화한 것이 아니라, 운영 흐름 전체를 정리한 것이 포인트입니다.

Before · 종이 출석

  • 종이에 서명, 필기체·누락으로 재확인 필요
  • 명단과 수기로 대조하며 인원 체크
  • 출석 정리·입력에 추가 공수 발생

After · PIN 출석

  • Class별 PIN을 공용 PC에서 입력
  • MS 연동으로 출석 시간·명단 자동 기록
  • 출석 누락 0건, 수기 정리 공수 감소

시연 방식

  • 실제 운영 화면을 공용 PC 2대로 녹화
  • 발표 시에는 링크 접속 없이 영상으로 흐름 시연
  • “이전 불편 → 개선 화면”이 한눈에 보이도록 구성

프롬프트 & 코파일럿 활용 팁

AI는 “완성품”이 아니라 “초안 + 검수 보조”로 쓸 때, 운영 품질이 가장 안전했습니다.

프롬프트 작성 3원칙

  • 맥락 먼저 – “교육 안내 메일 초안, 대상: 내부 임직원”
  • 형식 지정 – 제목/본문/버튼 문구 등 명확히 요청
  • 제한 조건 – 분량, 톤(존댓말), 금지 단어 등 명시

예) “SW 보안 세미나 개강 안내 메일 초안 써줘. 대상은 사내 개발자, 존댓말, 10줄 이내, 일정/장소/문의처는 {토큰}으로 남겨줘.”

코파일럿 활용 예시

  • HTML 코드 구조 점검·정리
  • 반복 문구를 토큰 형태로 분리할 때 아이디어 받기
  • Outlook/Forms에서 깨지는 부분 원인 확인
“완성품 제작”이 아니라,
초안 생성 + 검수 보조 역할로 사용하고 있습니다.

FAQ

요약

  • 코드를 몰라도 토큰만 작성하면 사용 가능
  • 오류 시에는 롤백·자동 체크 기능으로 대응
  • 다른 본부도 브랜드만 바꿔 바로 적용 가능

Q1. 코드를 몰라도 쓸 수 있나요?

A. 네. 기본 구조는 제가 세팅해 두고, 운영자는 토큰 값(과정명/일시/장소)만 입력하면 됩니다.

Q2. 오류 나면 어떻게 하나요?

A. 토큰 누락·오타는 발송 전 자동 체크하고, 템플릿 원본은 Teams에 보관해 바로 롤백이 가능합니다.

Q3. 다른 본부에서도 바로 쓸 수 있나요?

A. 과목·브랜드만 바꾸면 되도록 설계했습니다. 필요 시 교육운영팀에서 초기 세팅 지원이 가능합니다.

Q4. 유지보수 부담은?

A. 외부 시스템이 아니라, MS365 + HTML 코드 수준이라 비교적 가볍습니다. 템플릿을 한 번 수정하면 전체에 반영됩니다.
