Context

교육운영파트가 하는 일

교육 기획·운영, 안내, 출석, 결과 정리까지
작은 단위 업무들이 모여서 적지 않은 시간을 쓰고 있습니다.

주요 역할

  • 연간 다수의 교육과정 기획·운영
  • 수강신청, 안내 메일, 결과 보고 작성
  • 출석 관리 및 교육 이력 정리
  • 외부 강사 및 교육장 일정 조율

현장에서 느끼는 어려움

  • 비슷한 안내 메일·보고를 매번 새로 작성
  • 과정명·일시·장소를 옮겨 적다가 생기는 오타
  • 출석·결과 정리 작업이 행사마다 반복
  • 정작 기획·소통에 쓰는 시간은 부족
같은 일을 계속 반복하기보다는, 자주 쓰는 흐름을 한 번 정리해 두는 것이 이번 시도의 출발점이었습니다.

Problem & Goal

어떤 점을 줄이고 싶었는지

“늘 비슷한 일을 반복한다”는 피로를 줄이고,
안내·출석·보고의 기본 품질을 일정하게 가져가는 것이 목표였습니다.

현재 불편

  • 과정마다 안내문 서식·문체가 조금씩 달라 검수 시간이 길어짐
  • 출석 기록은 종이·엑셀을 왔다 갔다 하며 정리
  • 행사 전·후에 같은 내용을 여러 번 재입력

개선 방향

  • 한 번 입력한 정보를 메일·안내·출석에 함께 쓰기
  • 서식·톤을 맞춰 검수 부담을 줄이기
  • 새 시스템 도입보다는, 지금 있는 도구를 정리해서 사용
오늘은 이 목표를 바탕으로, 실제로 적용해 본 세 가지 흐름을 공유드립니다.

이번 파일럿에서 정리한 세 부분

메일·안내·출석이 각각 따로 놀지 않도록,
세 부분을 한 흐름으로 맞춰 보았습니다.

① 메일 HTML 템플릿

  • 과정명·일시·장소를 토큰으로 관리
  • 공지 메일·인쇄물이 같은 형태로 생성
  • Outlook/EP에서 깨지지 않도록 단순한 구조 유지

② 개강 안내 화면

  • 메일의 핵심 정보를 한 화면에 정리
  • 외부 강사·내부 교육생이 같은 내용을 봄
  • 모바일에서도 읽기 쉬운 단일 페이지 형태

③ PIN 출석

  • Class별 PIN으로 공용 PC에서 출석 입력
  • MS Forms / Power Automate와 연동하기 쉽게 설계
  • 출석 누락·수기 정리 시간을 줄이는 것이 목표
세 부분 모두 MS365 + 간단한 코드만으로 구현 가능한 수준에서 정리했습니다.

개선 1 – HTML 메일 템플릿

자주 쓰는 안내 구조를 HTML로 만들어두고,
과정별 정보만 바꿔서 쓰는 형태로 정리했습니다.

Before

  • 저장된 메일 템플릿을 불러와 과정명 등 수기로 편집
  • 대회 등의 안내문은 내용이 방대하여 폰트·여백·표현이 담당자마다 달라, 검수에 시간 소요
  • 메일·인쇄용 파일이 따로 놀아, 수정 시 이중 작업 발생

After

  • HTML 템플릿 1종에 토큰만 교체해서 사용
  • 메일·PDF·안내 페이지가 같은 문장과 구조를 사용
  • 작성+검수 시간이 크게 줄어듦

메일 화면 예시 (정리된 형태)

[BizTechON 교육운영] {과정명} 안내

안녕하십니까.
{과정명} 교육 일정을 아래와 같이 안내드립니다.

  • 일시 : {일시}
  • 장소 : {장소}
  • 대상 : {대상}

교육 당일에는 현장 출석을 위해 안내드린 PIN 번호를 준비해 주시면 됩니다.

출석 PIN : {PIN} (교육 시작 시 공용 PC에서 입력)

실제 운영 시에는 {과정명}, {일시}, {장소}, {PIN} 등에 값을 한 번만 넣고 재사용합니다.

개선 3 – PIN 기반 세미나 출결 (슬라이드 내 데모)

종이 서명 대신, 공용 PC 하나로 출석을 받는 화면입니다.
발표 도중 직접 이름·사번·PIN을 입력해 보는 데모로 사용하시면 됩니다.

출석 흐름

  • 운영자가 Class별 PIN을 미리 설정
  • 교육 시작 시, 공용 PC에서 출석 화면을 띄움
  • 교육생은 이름·사번·PIN만 입력하고 자리로 이동
  • 실제 환경에서는 MS Forms / Power Automate로 기록 연동
발표 시에는 이 화면에서 간단히 입력해 보고,
“실제 운영 환경에서는 이 데이터가 Forms로 넘어간다”고 설명하시면 좋습니다.

AI는 어디에, 어떻게 사용했는지

AI를 전면에 내세우기보다는,
초안 작성과 코드 정리에 조용히 도움을 받는 정도로 활용했습니다.

AI/코파일럿 활용 방식

  • 교육 안내 메일 초안 작성 (톤·분량은 직접 조정)
  • HTML 구조 점검 및 중복 코드 정리 아이디어
  • Forms / Outlook에서 깨지는 부분 원인 파악 도움
“완성품을 대신 만드는 도구”라기보다,
초안과 검수에 도움을 주는 도구 정도로 사용하는 것이 안정적이었습니다.

향후 정리해 보고 싶은 부분

  • 세미나 출석 화면과 MS Forms의 연동 고도화
  • 개강 안내 페이지 템플릿을 다른 과정에도 적용
  • 반복 보고 양식을 몇 가지 유형으로 통일

FAQ & Closing

자주 나오는 질문 & 마무리

FAQ

Q. 코드를 잘 몰라도 쓸 수 있을까요?
A. 기본 틀은 미리 만들어 두고, 운영자는 토큰 값(과정명·일시·장소·PIN)만 바꾸는 형태라 크게 어렵지 않습니다.
Q. 다른 본부·조직에서도 활용이 가능할까요?
A. 안내 문구와 디자인만 조금씩 손보면, 같은 구조를 그대로 쓸 수 있도록 구성했습니다.
Q. 유지보수 부담은 크지 않을까요?
A. 별도 솔루션이 아니라 MS365 + HTML 수준이라, 수정 시 한 번만 손보면 전체에 반영됩니다.
