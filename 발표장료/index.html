1

추진 배경

현장의 문제 인식

  • 교육 문서·메일 수기 작성 비중 높음
  • 서식 상이 → 검수 시간 증가
  • 출석·결과·보고 흐름 단절

필요성

  • 반복 업무 축소, 품질 일정화
  • 누가 해도 같은 결과
  • 한 번 입력해 전 단계 활용
“새 시스템보다 형식 정리부터.”
2

현장의 문제 인식

1. 반복 입력

  • 과정명·일시 매번 재입력
  • 복붙 중 오타

2. 불일치

  • 폰트/여백/문체 제각각
  • 문서 통일 어려움

3. 지연

  • 취합 리드타임 1~2일
  • 형식 수정 반복
작은 차이의 누적이 신뢰도에 영향.
3

개선 목표

단순화

중복 입력 → 자동 치환

일관성

메일·인쇄·보고 규칙 통일

연결성

공지→출석→결과→보고 데이터 연결

복잡한 자동화보다 정확한 흐름.
4

오늘의 데모 – 3가지

데모 1: HTML 템플릿 메일링

형식화된 템플릿 → 아웃룩 호환

깨짐 없는 메일, 품질 일정화

데모 2: 외부 강사 정보 자동화

입력 → Email 자동 발송

반복 엑셀 작업 제거, MS 연동 예정

데모 3: 웹 기반 출석 시스템

PIN 입력 → 자동 출석 기록

수기 출결 제거, 실시간 반영

반복 업무를 자동화하고, 연결성을 강화

5

표준 템플릿 3종 (메일 / 인쇄 / 보고)

메일

  • HTML 코드 기반 템플릿 → 아웃룩 호환
  • 버튼·스타일 고정
  • 깨짐 없는 메일링

인쇄

  • 고객사 맞춤 이미지 활용 가능
  • 망고보드 등 제한 없이 HTML 기반 제작
  • 운영자 별 본인 니즈에 맞는 디자인 구현 가능

보고

  • CSV 업로드 → 명단 자동 생성
  • 반복 작성 제거, 데이터 활용성 강화
  • 활용 방법

세미나 과정 출석 문제 개선

  • 교육생들이 종이에 수기 출결을 진행

현재

  • Class 별로 지정된 암호 입력하여 PIN 출석 체크 방식으로 전환
  • MS 연동되어 자동 출석 업데이트 기능(출석 시간&명단)

효과

  • 출석 누락 0건
