Context

교육운영파트가 하는 일

교육 기획·운영, 안내, 출석, 결과 정리까지
작은 단위 업무들이 모여서 적지 않은 시간을 쓰고 있습니다.

주요 역할

  • 연간 다수의 교육과정 기획·운영
  • 수강신청, 안내 메일, 결과 보고 작성
  • 출석 관리 및 교육 이력 정리
  • 외부 강사 및 교육장 일정 조율

현장에서 느끼는 어려움

  • 비슷한 안내 메일·보고를 매번 새로 작성
  • 과정명·일시·장소를 옮겨 적다가 생기는 오타
  • 출석·결과 정리 작업이 행사마다 반복
  • 정작 기획·소통에 쓰는 시간은 부족
같은 일을 계속 반복하기보다는, 자주 쓰는 흐름을 한 번 정리해 두는 것이 이번 시도의 출발점이었습니다.

Why

왜 이 Demo들을 만들었는가

  • 과정마다 안내 메일·방문 안내·출석 방식이 제각각
  • 같은 정보를 메일·문서·출석부에 여러 번 입력
  • 수정이나 일정 변경 시, 중복 수정·문의 폭증
올해는 “안내 → 방문 → 출석”의 흐름을 하나로 묶어,
교육운영파트의 반복 업무를 줄이는 데 초점을 맞췄습니다.

Problem

반복적인 업무 (개선 필요)

안내·커뮤니케이션

  • 과정마다 Word/PPT를 새로 열어 안내 메일 작성
  • 메일·안내문·인쇄물 문구가 달라 수정 시 이중 작업
  • “몇 시부터죠? 어디로 가면 되나요?” 문의 반복

출석·정리

  • 종이 출석부 서명 → 엑셀 재입력까지 행사당 10~15분
  • 사번 오기입·필기체 서명으로 재확인 필요
  • 한 번 적은 정보를 여러 시스템에 다시 입력
완전히 새로운 시스템을 만드는 대신,
지금 쓰는 도구(MS365 + 간단한 웹 페이지 + Copilot) 안에서 해결하는 것을 목표로 했습니다.

Demo 1 – 안내 메일, 구조부터 정리

왜 손을 댔는지

  • 과정마다 메일 형식·톤이 달라 검수 시간이 많이 소요
  • 메일·PDF·인쇄물 내용이 어긋나면 이중·삼중 수정
  • 작은 오타·날짜 오류를 잡는 데도 반복 공수 사용

어떻게 정리했는지

  • 고정 문장과 자주 바뀌는 값을 분리
  • 바뀌는 값은 {토큰}으로 표기 → 한 번만 입력
  • Outlook/EP 메일에서 바로 붙여넣을 수 있는 HTML 구조 유지

Demo 3 – PIN 출석, 종이 대신 공용 PC

기존 방식의 한계

  • 종이 출석부 서명 → 판독 어려운 필기체, 누락
  • 행사당 10~15분씩 엑셀 재입력·중복 체크
  • 추후 이력 반영 시 또 한 번 손으로 옮겨 적기

바꾸고 싶었던 것

  • 교육생이 이름·사번·PIN만 입력하면 출석 자동 기록
  • 운영자는 화면만 띄워두고, 세션 후 데이터만 확인
  • MS365/Power Automate와 자연스럽게 연동되는 구조

전체 흐름 속 위치

  • Demo 1 메일에서 이미 {PIN} 안내
  • Demo 2 방문 안내 페이지에서 현장 출석 방식 설명
  • Demo 3 화면을 공용 PC에 띄워, 교육생이 PIN으로 출석

AI · Copilot

Copilot, 이렇게 썼을 때 가장 도움이 됐습니다

주로 맡긴 일

  • 개강 안내 메일 초안 작성
  • 기존 안내문에서 토큰으로 묶을 부분 제안받기
  • 방문 안내 페이지 문구를 강사용/교육생용으로 구분 정리

실제 프롬프트 예시

  • [메일] “{과정명} 개강 안내 메일을 내부 임직원 대상, 8줄 이내로 써줘. 일정·장소·대상·문의처는 {토큰}으로 남겨줘.”
  • [토큰] “이 메일 내용에서 과정마다 바뀌는 값만 {토큰}으로 표시해줘.”
  • [리뷰] “아래 HTML 코드에서 모바일 깨질 부분과 불필요한 인라인 스타일을 정리해줘.”
Copilot이 내놓은 결과를 그대로 쓰기보다는,
“초안 생성 + 검수 도우미”로 사용하는 것이 가장 안정적이었습니다.
Copilot 대화 흐름 데모 (무한 반복)

Copilot과의 대화를 천천히 반복 재생하는 데모입니다.
